About The Course

The job market is completely independent of your skills here. I’ve seen Doctors from other countries come and start their restaurant as well as teachers coming in and joining NGOs.

What I’d suggest you is to rephrase the question. Out of the available jobs in IT, what kind of job am I capable enough of doing and how can I start working on it from now in order to gain enough exposure, before applying for jobs?

For example, You’ll meet a 100 people everyday who’ll say I can work on AI or I’m interested in learning AI (since AI is a boom in the market) but the question is, how many people do you actually encounter that say, I’ve worked on AI, this is what I’ve achieved so far- how can I learn and add more to it?

That’s the approach that’s going to get you in a position to land a job soon.

Course Overview

1. Introduction to Lean Six Sigma

  • History of Quality (Deming, Juran, JIT, Ishikawa, Taguchi, etc.)
  • Evolution of Six Sigma
  • Defining Six Sigma – philosophy and objectives
  • Overview of Six Sigma DMAIC process

2. Stakeholders & Setting up a Lean Six Sigma Project

  1. Identifying and documenting stakeholder requirements
    1. Identifying stakeholders and customers
    2. Data collection and analysis
    3. Determining critical requirements
  2. Selection Criteria Project
    1. Identifying performance metrics
    2. Using financial criteria to evaluate project benefits
    3. Maximizing project benefits for the organization
  3. Project Planning
    1. Creating Project Charter
    2. Charter Negotiation
  4. Managing Team Dynamics
    1. Initiating teams
    2. Stages of team evolution
    3. Maslow’s hierarchy of needs
    4. Motivation Techniques
    5. Conflict Resolution Techniques
    6. Management / Leadership styles
    7. Roles played by people in a project
  5. Important project management & planning tools

3. Lean Six Sigma Methodology – Define

  1. Inputs – Need for Six Sigma project, Executive management sponsorship, core team Identified
  2. Tools
    1. Organization hierarchy
    2. High level process maps
    3. High level Pareto charts
    4. Idea generation and categorization tools
  3. Outputs
    1. Project charter
    2. Established metrics
    3. Problem statement
    4. Roles & responsibilities

4. Lean Six Sigma Methodology – Measure

  1. Objectives of Measure Phase
  2. Inputs – the outputs of the Define phase
  3. Tools
    1. Data collection tools and techniques
    2. Measurement scales
    3. Validation techniques (Gauge R & R)
    4. Statistical distributions
    5. Data mining
    6. Run charts
    7. Process map
    8. Stakeholder tools
    9. Process costs
  4. Outputs
    1. Well defined processes
    2. Baseline process capability
    3. Process parameters affecting CTQs
    4. Cost of poor quality (COPQ)
    5. Measurement system

5. Lean Six Sigma Methodology – Analyze

  1. Objectives of Analyze Phase
  2. Inputs – outputs of the Measure phase
  3. Tools
    1. Ishikawa diagram
    2. Failure mode and effects analysis
    3. Hypothesis testing
    4. Process capability study
  4. Outputs
    1. Important causes of defects
    2. Special and common causes of variation
    3. DPMO and sigma level

6. Lean Six Sigma Methodology – Improve

  1. Objectives of Improve Phase
  2. Inputs – outputs of the Analyze phase
  3. Tools
    1. Returns on investment
    2. Solution design matrix
    3. Design of experiment
    4. Taguchi robustness concepts
    5. Response surface methodology
    6. Project planning and management tools
    7. Prototypes
  4. Outputs
    1. Cost / benefit for different solution
    2. Selection of solutions for implementation
    3. Implementation plan

7. Lean Six Sigma Methodology – Control — Health Care Domain Concepts

  1. Objectives of Control Phase
  2. Inputs – outputs of the Improve phase
  3. Tools
    1. Control plan
    2. Statistical process control
    3. Lean enterprise
    4. 5S
    5. Kaizen
    6. Kanban
    7. Total productive maintenance
    8. Measurement system reanalysis
  4. Outputs
    1. Implemented solutions
    2. Revised measurement system
    3. Control plan for sustaining benefits
    4. Improves process capability
    5. Lessons learned

8. More on Lean

  1. Lean is speed
  2. Value stream map
  3. Total supply chain
  4. Lean six sigma logistics
  5. Standard operations
  6. Operator work instructions
  7. Cycle time reduction and talk time
